Responsive mobile — espace compte
Navigation mobile complète, pagination des annonces, correctifs de layout. 8 fichiers modifiés, 2 créés.
UX
Navigation mobile — barre fixe + tiroir slide-up Nouveau composant MobileNav.tsx rendu hors du conteneur flex (indépendant du layout). Barre fixe 56px en bas (Accueil / Annonces / Messages / Visites / Menu). Bouton Menu ouvre un tiroir slide-up avec profil, tous les liens, "Publier" et déconnexion. Tout en style={{}} inline pour éviter tout problème de génération JIT Tailwind. Masquage/affichage via @media dans une balise <style> directement dans le layout server — fiable à 100%, pas de dépendance au cache HMR.
UX
Sidebar desktop séparée (CompteNav) Refactoring : CompteNav ne gère plus que la sidebar desktop (classe compte-sidebar). Séparation stricte des deux composants pour éviter le bug Fragment-in-flex qui empêchait les éléments position:fixed de s'ancrer correctement au viewport.

Tiroir mobile — bord supérieur visible quand fermé Bug géométrique : tiroir à bottom: 56px + translateY(100%) laissait le bord supérieur visible au-dessus de la barre. Corrigé en positionnant à bottom: 0 + translateY(110%) + paddingBottom: 56px interne.

Page statistiques : grilles xl:grid-cols-* → lg:grid-cols-*, padding réduit mobile, boutons header adaptés. 30 avril 2026
Correctif tracking des vues
Les vues des annonces n'étaient jamais enregistrées — deux bugs combinés identifiés et corrigés.
Fix
Table vues_stats et RPC increment_vue_stat manquants La table vues_stats (historique journalier des vues par bien) et la fonction RPC increment_vue_stat étaient appelées dans le code mais n'existaient jamais en base. Migration 20260430_vues_stats.sql créée : table avec contrainte unique (bien_id, date), index, RLS lecture publique, fonction upsert security definer.
Fix
Promesse abandonnée — passage à after() Next.js 15 Le pattern (async () => { ... })().catch(() => {}) dans la page /annonce/[id] était une promesse non-awaitée : Next.js App Router l'abandonnait avant exécution dès que la réponse était envoyée. Remplacé par after() de next/server, qui exécute le callback après l'envoi de la réponse sans bloquer le rendu.

29 avril 2026 — session 3
Audit de sécurité complet
Correction de 8 vulnérabilités identifiées : XSS, rate limiting, validation des inputs, headers HTTP, RGPD.
Sécu
XSS corrigé dans MapCanvas — données Overpass API Ajout d'une fonction esc() locale et application sur p.name et p.subtype dans les 3 occurrences innerHTML/setHTML alimentées par l'API externe Overpass.
Sécu
Rate limiting email — max 5 messages/heure/IP Création de src/lib/emailRateLimit.ts et de la table Supabase email_rate_limits (avec RLS). Appliqué sur les 4 routes : contact-agent, chercheur-contact, contact, publicite-contact. Retourne 429 si dépassé.
Sécu
Validation & sanitisation dans contact-agent Longueurs max (nom 100, email 254, téléphone 20, message 2000), validation regex email, trim des champs, escHtml sur toutes les variables injectées dans le HTML de l'email.
Sécu
STRIPE_WEBHOOK_SECRET — suppression du non-null assertion Remplacement de process.env.STRIPE_WEBHOOK_SECRET! par une vérification explicite retournant un 500 clair si la variable est absente.
Sécu
updateProfil — escalade de privilèges via impersonation bloquée Si viewId !== user.id (impersonation active), vérification que l'appelant est admin avant toute mise à jour de profil.
Sécu
Headers HTTP — COOP & CORP ajoutés Ajout de Cross-Origin-Opener-Policy: same-origin et Cross-Origin-Resource-Policy: same-site dans next.config.ts. COEP omis (incompatible MapLibre — tuiles cross-origin).
RGPD
Anonymisation IP dans map_ad_events Fonction anonymizeIp() : IPv4 → dernier octet à 0 (ex: 192.168.1.0), IPv6 → 48 bits conservés. Appliquée avant tout stockage ET avant les comparaisons de rate limiting et déduplication.
Fix
Suppression de proxy.ts dupliqué — middleware edge déjà actif Le fichier proxy.ts (convention Vercel) appelait déjà updateSession qui protège /admin/* et /compte/* au niveau edge. Le middleware.ts créé par erreur a été supprimé.
